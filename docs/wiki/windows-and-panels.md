# ウィンドウとパネル

中央領域はドラッグ可能なフローティングウィンドウ群で構成されます。各ウィンドウは独立して移動・前面化・クローズでき、レイアウトは保存されます。関連ページ: [画面レイアウト](screen-layout.md) / [チャート](chart.md) / [戦略エディタ](strategy.md) / [ファイルメニュー](file-menu.md)。

> 文中の `[A1]` などは、その挙動を保証する E2E flow の ID。一覧は [`tests/e2e/FLOWS.md`](../../tests/e2e/FLOWS.md) を参照。

## 共通操作

すべてのフローティングウィンドウは同じ枠構造（タイトルバー＋本体）を共有し、以下の操作が共通です。

| 操作 | 方法 |
| --- | --- |
| 移動 | タイトルバーをドラッグ [M2] |
| 前面化（z オーダー） | ウィンドウ本体をクリック [M4] |
| 閉じる | タイトルバー右端の `×` ボタン [M3]（Startup は例外で `×` を持ちません [M7]） |
| 開く | サイドバーの Panels セクションのボタン [M1]（Startup は例外で実行モードが表示を制御します [M8]） |

- タイトルバーは上端の濃色バーで、左端にウィンドウタイトル、右端に赤い `×` ボタンが付きます。 [M3]
- 同じ種類のパネルは原則 1 つだけ開きます（戦略エディタは複数開けます）。 [M5]
- ウィンドウを移動すると、レイアウトは約 1 秒のデバウンス後に自動保存されます（[ファイルメニュー](file-menu.md) のレイアウト永続化を参照）。移動・クローズは Undo / Redo の対象です。 [M2]/[M3]/[I11]
- **例外: Startup ウィンドウ**（下記）は `×` を持たず、Panels ボタンからも開きません。表示・非表示は実行モードが所有します。 [M7]/[M8]

## Buying Power（バイイングパワー）

- アクセント色: シアン。サイズ: 約 270 × 130。タイトル: `BUYING POWER`。 [M1]/[L4]
- バックエンド由来のポートフォリオ状態を表示する読み取り専用パネルです。Replay 完了後の反映は [B1]、ライブ口座更新（AccountEvent）は [F4]。

| 行ラベル | 内容 |
| --- | --- |
| `equity:` | 評価総額。値が 0 以上で緑、負で赤。 |
| `cash:` | 現金残高。 |
| `BP:` | バイイングパワー（買付余力）。 |

- 値が未ロードのときは各行が `—` を表示します。

## Positions（ポジション）

- サイズ: 約 280 × 200。タイトル: `POSITIONS`。最大 5 行まで表示します。 [M1]/[L4]
- 読み取り専用の保有ポジション一覧です。Replay 完了後の反映は [B1]、ライブ口座更新は [F4]。

| 列ヘッダ | 内容 |
| --- | --- |
| `Sym` | 銘柄シンボル |
| `Qty` | 数量。0 以上で緑、負で赤。 |
| `Avg` | 平均取得価格 |
| `uPnL` | 含み損益。0 以上で緑、負で赤。 |

- 状態メッセージ: 未ロード時は `No run yet`、ポジションが空のときは `No positions`。

## Orders（オーダー）

- サイズ: 約 360 × 220。タイトル: `ORDERS`。最大 6 行まで表示します。 [M1]/[L4]
- 読み取り専用の注文一覧です。Replay 完了後の反映は [B1]、ライブ注文イベント（OrderEvent）は [F3]、注文 seed は [H1]。

| 列ヘッダ | 内容 |
| --- | --- |
| `Sym` | 銘柄シンボル |
| `Side` | 売買区分。`BUY` で緑、`SELL` で赤、その他はグレー。 |
| `Qty` | 数量 |
| `Price` | 価格 |
| `Status` | 注文ステータス |

- 状態メッセージ: 未ロード時は `No run yet`、注文が空のときは `No orders`。

## Run Result（実行結果）

- サイズ: 約 280 × 160。タイトル: `RUN RESULT`。
- 戦略実行（[バックテスト](backtest.md) / [リプレイ](replay.md)）の結果を表示する読み取り専用パネルです。4 行構成です。
- **他のウィンドウと異なる点**:
  - `×` クローズボタンを持ちません（ユーザーは閉じられません）。 [M13]/[M17]
  - Replay / LiveAuto のときだけ表示され、LiveManual では非表示になります。可視性は実行モードが所有し、サイドバーの Panels ボタンからは開きません。実行モードが切り替わった同じフレームで表示/非表示が追従します。 [M14]/[M15]/[M16]/[M17]
  - 起動時に自動 spawn されます（サイドバーに手動 open ボタンはありません）。 [M15]/[M17]

| 行 | 内容 | E2E |
| --- | --- | --- |
| State | 実行状態。`No run yet`（グレー）/ `Running…`（黄）/ `Completed`（緑）/ `Failed: {error}`（赤）。 | Running→Completed [A1] / Failed [A6] |
| Run ID | `run: {id}` 形式。実行 ID 未設定時は空。 | [A1] |
| Stats | `fills: {件数}  eq_pts: {点数}`。約定件数とエクイティ点数。サマリ未取得時は空。 | [B2] |
| P&L | `pnl: {損益}`。0 以上で緑、負で赤。サマリ未取得時は空。 | [B2] |

## Strategy Editor（戦略エディタ）

- アクセント色: 紫。サイズ: 約 500 × 400。 [M1]/[L4]
- cosmic-edit ベースのコードエディタで、戦略 `.py` を編集します。複数のエディタウィンドウ（リージョン）を同時に開けます。 [M5]/[J1]
- **Manual モードでは非表示**です。エディタウィンドウとサイドバーの「Strategy Editor」ボタンが隠れます（Replay / Auto では表示）。これは Manual 中だけの**一時的な上書き**で、Manual を抜けると元の可視性へ復元します。Startup と異なり、保存された layout の `visible` は**権威のまま**（save/restore 方式で一時 hide が永続 layout に焼き込まれないようにしています）。モード切替は backend から受けた同一フレームで追従します。 [M12]/[M19]
- 詳細な編集機能（Find / Replace、リージョン分割、シンタックスハイライト等）は [戦略エディタ](strategy.md) を参照してください。

## Order（発注フォーム）

- LiveManual 限定のサイドバー **Order** ボタンから開く、ワールド空間のスプライト製フローティングウィンドウ（タイトル `ORDER`）です。サイドバー選択中の銘柄に発注します。 [K10]
- ドラッグで移動でき、パン（カメラ移動）に追随し、`×` で閉じます。LiveManual を離れると despawn し、レイアウト永続化の対象外です。 [K10]
- フォーム各ボタンはスプライト + `Pointer<Click>` observer で `OrderButtonPressed` イベントを駆動します（Bevy UI Node ではありません）。 [K10]
- side（BUY / SELL）・order type・数量・価格を指定し、`[発注]` で validation → 2 段階 confirm モーダル → `[Confirm]` で `PlaceOrder`。確認モーダルは中央オーバーレイの UI Node のままです。 [K7]/[K10]/[K11]
- 注文行の右クリック context menu から訂正・取消（訂正モーダル）を開始します。 [K9]/[K12]/[K16]
- Tachibana で第二暗証番号を要求されると SecretModal が開きます。 [F5]/[K8]/[K15]
- 詳細は [注文](orders.md) を参照してください。

## Live Run Panel（ライブラン）

- Auto モードで起動した Live Auto run を一覧表示し、`[Pause]` / `[Resume]` / `[Stop]` で制御するパネルです（Bevy UI Node + Interaction 流派、run が無いあいだは非表示）。 [N1]
- 各 run 行に lifecycle 状態（READY / RUNNING / PAUSED / STOPPING / STOPPED / ERROR、色分け）・開始時刻・run 単位の PnL / order / fill テレメトリを表示します。 [N1]/[N2]
- 下部に戦略ログ（`self.log.*`）の直近行を tail 表示します（リングバッファ 100 行）。 [N4]
- Safety Rail 違反は Footer のトーストで通知されます。 [N3]

## Startup（実行条件）

- アクセント色: 青。サイズ: 約 260 × 200。タイトル: `STARTUP`。Replay モードのときだけ表示されます。 [M8]
- リプレイの実行条件（開始日・終了日・粒度・初期資金）を編集するフローティングウィンドウです。フィールド検証・編集の cache 書き戻しは [J7]/[J8]/[J16]。詳細は [Replay 実行](replay.md) を参照してください。
- **他のウィンドウと異なる点**:
  - `×` クローズボタンを持ちません（ユーザーは閉じられません）。 [M7]
  - サイドバーの Panels ボタンからは開きません。表示・非表示は実行モードが所有します（Replay のみ可視）。 [M8]
  - タイトルバーをドラッグして移動でき、位置・z はレイアウトに永続化されますが、保存された `visible` は復元時に無視されます（可視性は常に実行モードが決める）。 [M9]

## Chart（チャート）

- サイズ: 約 360 × 244（Live モードでは Ladder ペインを伴い拡大）。タイトル: `CHART — {instrument_id}`。 [K1]/[K5]
- ローソク足チャートを表示します。詳細は [チャート](chart.md) を参照してください。 [K1]
- チャートはサイドバーで選択中の銘柄から自動的に導出され、パネルとしてのレイアウト保存対象外です。 [J13]/[K1]/[I7]

## 表示の前提（重要）

- Buying Power / Positions / Orders / Run Result はいずれもバックエンド由来の読み取り専用表示です。 [B1]/[F3]/[F4]
- これらのパネルは Replay 実行の結果反映に加え、Live（Manual / Auto）の注文イベント（OrderEvent）・口座更新（AccountEvent）でもリアルタイムに更新されます。 [F3]/[F4]/[H1]/[H2]
- 発注・訂正・取消の操作は読み取り専用パネルではなく **Order** ウィンドウから行います。Live Auto 戦略の制御は **Live Run Panel** です。
