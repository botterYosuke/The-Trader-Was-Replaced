# Bevy 0.15 vs 0.19-dev チートシート

このプロジェクトは **0.15** で固定。`.claude/skills/bevy-engine/src/` は **0.19-dev**。
ミラーで見た書き方をそのまま貼ると落ちることがある。落ちたらここを引く。

## ECS / World

| 操作 | 0.15 | 0.19-dev |
|---|---|---|
| 親子参照 component | `Parent` | `ChildOf` |
| 親 entity を取り出す | `parent.get()` (returns `Entity`) | `child_of.parent()` |
| `Query` から単一 entity | `q.get_single()` → `Result<&T, _>` | `q.single()` → `&T` (panic) or `q.single().ok()` |
| `Query` から単一 mut | `q.get_single_mut()` | `q.single_mut()` |
| `Time::delta_seconds` | `time.delta_seconds()` (f32) | `time.delta_secs()` |
| `Time::elapsed_seconds` | `time.elapsed_seconds()` | `time.elapsed_secs()` |
| Observer の対象 entity | `trigger.entity()` (method) | `trigger.target()` |
| Observer の event 本体 | `trigger.event()` (method) | `trigger.event()` （変わらず） |

## Rendering / Sprite / Text

| 0.14 以前 | 0.15 (このプロジェクト) | 0.19-dev |
|---|---|---|
| `SpriteBundle { sprite, transform, .. }` | `(Sprite { color, custom_size, ..default() }, Transform::from_xyz(...))` | 同 0.15 |
| `Camera2dBundle::default()` | `Camera2d` | `Camera2d` |
| `Text2dBundle { text, transform, .. }` | `(Text2d::new(s), TextFont { font_size, .. }, TextColor(c), Transform::from_xyz(...))` | 同 0.15 |
| `Color::rgb(r,g,b)` | `Color::srgb(r,g,b)` | `Color::srgb(r,g,b)` |
| `Color::rgba(r,g,b,a)` | `Color::srgba(r,g,b,a)` | 同左 |

**Camera2d の Projection 取り出し**:
- 0.15: `Query<&OrthographicProjection, With<Camera2d>>` → `.get_single().map(|p| p.scale)`
- 0.19: `Projection` enum 経由になっている箇所がある（`Projection::Orthographic(p)` を match）

## Hierarchy / Spawn

```rust
// 0.15 ✅
let parent = commands.spawn(SpatialBundle::default()).id(); // SpatialBundle は 0.15 でまだ生存
let child  = commands.spawn(Transform::default()).id();
commands.entity(parent).add_child(child);
// 取得: parent_q.get(child).map(|p| p.get())

// 0.19-dev では SpatialBundle 廃止。Transform + Visibility をタプルで:
let parent = commands.spawn((Transform::default(), Visibility::default())).id();
// hierarchy: ChildOf component が自動で挿入される設計に移行
```

このプロジェクトは Transform 直 spawn でやっていることが多い（`spawn_floating_window`）。
`SpatialBundle` を見たら 0.14/0.15 系。`ChildOf` を見たら 0.19。

## Required Components

0.15 で導入された機能。`#[require(...)]` で「この component があるなら必ずこれも」と
書ける。`Sprite` には `Transform` と `Visibility` が required で付くため、本来は
`Sprite` だけ spawn しても動く。しかし本プロジェクトは **明示タプル spawn** を採用
している（位置決めしたいので Transform を必ず指定する）。それでいい、合わせる。

## Picking / Pointer (0.15)

```rust
use bevy::prelude::*;

commands.spawn((
    Sprite { ..default() },
    Transform::from_xyz(0., 0., 0.),
))
.observe(|trigger: Trigger<Pointer<Down>>,
          mut q: Query<&mut Transform>| {
    let entity = trigger.entity();           // 0.15: method
    let event  = trigger.event();             // PointerEvent
    // event.button, event.pointer_location など
});
```

`bevy_picking` の `bevy_sprite_picking_backend` feature が有効なので Sprite は
そのまま pickable。`Cargo.toml` の `features = ["default", "bevy_picking", "bevy_sprite_picking_backend"]`
を確認。

0.19 では `trigger.entity()` → `trigger.target()` と読み替える。

## Event API

0.15:
```rust
#[derive(Event)] struct Foo(pub i32);
app.add_event::<Foo>();
fn writer(mut w: EventWriter<Foo>) { w.send(Foo(1)); }
fn reader(mut r: EventReader<Foo>) { for e in r.read() { /* ... */ } }
```

0.19 では `EventWriter::send` が `EventWriter::write` に名称変更されつつある
（実際には deprecated alias が並走している期間）。0.15 では `send` のままで OK。

## Schedule / Systems

0.15: `app.add_systems(Update, (a, b, c))` — タプル並列、`.chain()` で順序、
`.run_if(condition)` でゲート、`.after(other_sys)` / `.before(...)` で順序制御。
これは 0.19 でも同じ。

`Startup` / `Update` / `PreUpdate` / `PostUpdate` / `FixedUpdate` 等のスケジュール
ラベルも同じ。`First` / `Last` もある。
